### Task 1: 偏好存储封装
**type:** implementation

创建 `src/lib/preferences.ts`：

1. **localStorage 结构:**
   ```
   key: "dexter-preferences"
   value: JSON.stringify({
     model: "gpt-5.4",
     provider: "openai",
     theme: "light" | "dark" | "system",
     apiKeys: {
       openai: "sk-...",
       anthropic: "sk-ant-...",
       google: "...",
       xai: "...",
       moonshot: "...",
       deepseek: "..."
     }
   })
   ```

2. **API:**
   ```
   loadPreferences() → Preferences
   savePreference(key, value) → void
   getApiKey(provider) → string | null
   setApiKey(provider, key) → void
   getTheme() → 'light' | 'dark' | 'system'
   setTheme(theme) → void
   ```

3. **默认值:**
   - model: "gpt-5.4"
   - provider: "openai"
   - theme: "light"（保持与现有 warm cream 配色一致）
   - apiKeys: 全部 null

**验证:** save/load 循环正确，默认值合理

**Files to create:**
- `src/lib/preferences.ts` (新建)

### Task 2: API Key 管理与模型选择器对接
**type:** implementation

修改模型选择器和会话创建流程，接入 API key：

1. **模型选择器改造 (`src/components/model-selector.tsx`):**
   - 切换 provider 时，检查是否有对应 API key
   - 如果没有 key，在模型列表下方显示提示："未设置 API key，请在设置中配置"
   - 不阻塞切换操作（key 可能通过其他方式提供，如环境变量）

2. **Session 创建对接 API key:**
   - 修改 `useChatSession.createNewSession()`
   - 创建 session 时，从 preferences 读取当前 provider 的 API key
   - 如果有 key，在 POST body 中传递: `{ model, provider, apiKey }`
   - 服务端 `POST /api/runtime/sessions/route.ts` 接收 apiKey，用于 LLM 调用

3. **服务端 API key 处理 (`src/app/api/runtime/sessions/route.ts`):**
   - 如果请求中有 apiKey，传入 createWebRuntimeSession 或 adapter 配置
   - 如果没有 apiKey，使用服务端环境变量中的 key（现有行为不变）
   - apiKey 不持久化到服务端内存，每次请求从客户端传递

4. **模型切换时持久化:**
   - `useChatSession.changeModel()` 调用时，同时 `savePreference('model', model)` 和 `savePreference('provider', provider)`
   - 新建会话时从 preferences 读取上次选择的模型

**验证:** 设置 API key → 切换 provider → 创建新会话 → 确认使用对应 key

**Files to modify:**
- `src/components/model-selector.tsx` (改造)
- `src/hooks/use-chat-session.ts` (改造)
- `src/app/api/runtime/sessions/route.ts` (改造)

### Task 3: 暗色/亮色主题切换
**type:** implementation

实现主题切换功能：

1. **CSS 变量方案:**
   - shadcn/ui 已原生支持 dark mode
   - 在 `<html>` 或 `<body>` 上添加 `class="dark"` 切换暗色模式
   - 使用 Tailwind CSS 的 `dark:` 变体

2. **暗色模式色值映射 (在 03-UI-SPEC.md warm cream 配色基础上):**
   - 背景主色: `hsl(220 20% 10%)` — 深墨蓝（替代 warm cream）
   - 文本主色: `hsl(40 20% 90%)` — 暖白
   - 卡片背景: `hsl(220 15% 14%)`
   - 强调色: 保持 forest green `hsl(152 28% 29%)`
   - 错误色: 保持 red-600

3. **主题 hook (`src/hooks/use-theme.ts`):**
   ```
   useTheme() → { theme, setTheme, resolvedTheme }
   ```
   - theme: 'light' | 'dark' | 'system'
   - resolvedTheme: 'light' | 'dark'（system 解析后的实际值）
   - 监听 `prefers-color-scheme` 媒体查询
   - setTheme 时同步到 localStorage preferences + DOM class

4. **应用主题:**
   - 在 `src/app/layout.tsx` 或 `src/components/workspace-shell.tsx` 顶层初始化
   - 首次加载时从 preferences 读取主题，避免闪烁（可在 `<html>` 标签上内联 script）

**验证:** 切换到暗色 → 页面样式正确 → 刷新 → 保持暗色 → 切换到 system → 跟随系统

**Files to create/modify:**
- `src/hooks/use-theme.ts` (新建)
- `src/app/globals.css` (添加暗色 CSS 变量)
- `src/app/layout.tsx` (初始化主题，防止闪烁)

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| API key 明文存储安全 | Low | v1 本地单用户，无公网暴露。文档中注明风险 |
| 暗色模式色值不协调 | Medium | 在 globals.css 中统一定义，不散落在组件中 |
