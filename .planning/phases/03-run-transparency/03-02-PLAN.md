# 03-02: 停止、重试与继续控制

## Objective

让用户在流式进行中停止 agent 运行，停止后保留已生成内容并标记为"已中止"，用户可以继续发新消息或重新发送。实现 `POST /abort` 端点和服务端 AbortController 集成。

## must_haves

- [ ] 流式中发送按钮变为红色 Stop 按钮，点击触发 abort
- [ ] 新增 `POST /api/runtime/sessions/:id/abort` 端点
- [ ] 服务端 `runWebSession` 创建 AbortController 并通过 session registry 注册
- [ ] 中止后 assistant 消息标记 `status: 'aborted'`，内容保留
- [ ] 底部恢复发送按钮，用户可发新消息

<read_first>
- src/webui/runtime/adapter.ts (runWebSession 函数)

- src/webui/runtime/types.ts (WebRuntimeSession)

<action>
1. 在 `src/webui/runtime/types.ts` 中扩展 `WebRuntimeSession`:
   - 新增字段: `abortController?: AbortController`

2. 修改 `src/webui/runtime/adapter.ts` 的 `runWebSession`:
   - 在调用 `Agent.create` 之前创建 `AbortController`
   - 将 `abortController.signal` 传入 AgentConfig 的 `signal` 字段
   - 将 `session.abortController = abortController`
   - 在 `catch` 块中检测 `error.name === 'AbortError'`，此时设置 `session.status = 'aborted'`（而非 'error'）
   - 在 `finally` 块中清空 `session.abortController = undefined`

3. 创建 `src/app/api/runtime/sessions/[id]/abort/route.ts`:
   - POST 端点，无请求体
   - 通过 `getSession(id)` 找到 session
   - session 不存在 → 404
   - `session.abortController` 不存在 → 409（没有正在运行的操作）
   - 调用 `session.abortController.abort()`
   - 返回 `{ ok: true, status: 'aborted' }`

4. 修改 chat route:
   - 在 `runWebSession` 调用前确保 session 状态从 'idle' 改为 'running'
   - 这在 adapter 中已通过 `session.status = 'running'` 处理，确认无遗漏
</action>

<acceptance_criteria>

- `runWebSession` 创建 AbortController 并传入 AgentConfig.signal
- `POST /api/runtime/sessions/:id/abort` 端点存在
- abort 端点调用 `controller.abort()` 后 agent 循环被中断
- 中断后 `session.status` 为 `'aborted'`（不是 'error'）
- `tsc --noEmit` 无新增错误
</acceptance_criteria>

<action>
1. 在 `src/hooks/use-chat-session.ts` 中:
   - 新增 `abortSession` 异步函数:
     ```typescript
     const abortSession = async (): Promise<void> => {
       if (!sessionIdRef.current || !streamingRef.current) return;
       try {
         await fetch(`/api/runtime/sessions/${sessionIdRef.current}/abort`, { method: 'POST' });
       } catch { /* abort 请求失败不影响 UI 状态 */ }
     };
     ```
   - 在 `onComplete` 回调中: 如果消息 status 不是 'complete'（可能是 aborted），设为 `'aborted'`
   - 扩展返回对象，加入 `abortSession`
   - 扩展 `ChatMessage.status` 类型: `'streaming' | 'complete' | 'aborted'`（在 use-sse-stream.ts 中）

2. 修改 `src/components/chat-input.tsx`:
   - 新增 prop: `onAbort?: () => void`
   - 当 `disabled=true`（即 isStreaming）时:
     - 隐藏 Send 按钮和 Textarea
     - 显示 Stop 按钮: `<Button variant="destructive" onClick={onAbort}><Square />Stop</Button>`
   - Stop 按钮使用 `Square` icon (lucide-react)
   - 点击后调用 `onAbort()`，然后 `setInput('')` 清空输入

3. 修改 `src/components/workspace-shell.tsx`:
   - 从 `useChatSession` 解构 `abortSession`
   - 传给 `ChatInput`: `<ChatInput onSend={sendQuery} onAbort={abortSession} disabled={isStreaming} />`

4. 在 `src/components/chat-message.tsx` 中处理 `aborted` 状态:
   - 当 `message.status === 'aborted'` 时，在已有内容下方渲染中止标记:
     - `border-t border-border/60 pt-3 mt-3`
     - 文本: "已中止"（`text-xs font-medium text-muted-foreground uppercase tracking-wider`）
     - 提示: "发送新消息或重新发送"（`text-xs text-muted-foreground`）
   - 不显示 loading spinner（和 streaming 状态的区别）
</action>

## Threat Model

- **Mitigation:** v1 本地单用户部署。Session ID 使用 UUID，不易猜测
- **Risk level:** LOW（本地部署）
