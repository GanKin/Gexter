# 03-01: 渲染工具调用详情和审批流

## Objective

让 WebUI 展示工具调用的可展开详情（参数、返回值、耗时），并实现内联审批卡片交互。当 agent 需要用户批准敏感工具（`write_file`、`edit_file`）时，聊天流中嵌入审批卡片，用户点击后通过 POST 端点将决策发回服务端。

## must_haves

- [ ] 工具 Badge 点击后展开详情面板（参数、返回值预览、耗时）
- [ ] 审批卡片出现在聊天流中，显示工具名 + 参数摘要 + 三按钮
- [ ] 服务端在 `runWebSession` 中实现 `requestToolApproval` 回调，通过 Promise 等待浏览器 POST
- [ ] `POST /api/runtime/sessions/:id/approve` 端点接收审批决策
- [ ] SSE 流推送 `tool_approval` 事件（审批结果通知）

<action>
1. 修改 `src/webui/runtime/adapter.ts` 的 `runWebSession`:
   - 在 `Agent.create` 调用前创建 `AbortController`，将 `signal` 传入 config
   - 实现 `requestToolApproval` 回调:
     ```typescript
     requestToolApproval: async (request) => {
       const requestId = crypto.randomUUID();
       return new Promise<ApprovalDecision>((resolve) => {
         session.pendingApproval = { resolve, requestId };
       });
     }
     ```
   - 将 `session.approvedTools` 传入 `sessionApprovedTools`

2. 创建 `src/app/api/runtime/sessions/[id]/approve/route.ts`:
   - POST 端点，接收 `{ decision: 'allow-once' | 'allow-session' | 'deny', requestId: string }`
   - 通过 `getSession(id)` 找到 session
   - 验证 `session.pendingApproval` 存在且 `requestId` 匹配
   - 如果 decision 是 `'allow-session'`，将工具名加入 `session.approvedTools` Set
   - 调用 `session.pendingApproval.resolve(decision)` 解除 Promise 阻塞
   - 清空 `session.pendingApproval = null`
   - 返回 `{ ok: true }`
   - 错误情况: session 不存在 → 404，无待审批 → 409，requestId 不匹配 → 400

3. 修改 chat route (`src/app/api/runtime/sessions/[id]/chat/route.ts`):
   - 将 `PHASE2_EVENT_TYPES` 替换为扩展后的事件类型 Set（包含 tool_approval 和 tool_denied）
   - 将 `runWebSession` 的 config 中传入 `requestToolApproval` 和 `sessionApprovedTools`（adapter 已处理）
</action>

## Threat Model

- **Attack vector:** 审批端点可被其他标签页/脚本调用（CSRF）
- **Mitigation:** v1 本地部署模型，攻击面限于本机。Phase 4 可加 CSRF token
- **Risk level:** LOW（本地部署，无公网暴露）
