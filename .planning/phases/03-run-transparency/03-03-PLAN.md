# 03-03: 模型/Provider 选择器

## Objective

在左侧 sidebar 替换静态 Model 信息区为交互式两级选择器（先选 provider 再选 model），切换模型不重建 session，只更新 session.model 供下次 chat 调用使用。

## must_haves

- [ ] 侧边栏 Model 区域变为可点击的下拉选择器
- [ ] 第一级显示 6 个 provider（openai、anthropic、google、xai、moonshot、deepseek）
- [ ] 第二级显示选中 provider 的 model 列表
- [ ] 选择模型后更新 session 的 model 字段
- [ ] 下次 chat 调用自动使用新模型

<action>
1. 在 `src/hooks/use-chat-session.ts` 中:
   - 新增 state: `currentModel: string`（初始值从 health 端点获取，或默认 'gpt-5.4'）
   - 新增 `changeModel` 函数:
     ```typescript
     const changeModel = async (model: string, provider: string): Promise<void> => {
       if (!sessionIdRef.current) return;
       try {
         await fetch(`/api/runtime/sessions/${sessionIdRef.current}/model`, {
           method: 'PATCH',
           headers: { 'Content-Type': 'application/json' },
           body: JSON.stringify({ model, provider }),
         });
         setCurrentModel(model);
       } catch { /* 静默失败，本地 UI 已更新 */ }
       setCurrentModel(model);
     };
     ```
   - 从 health 端点响应中提取当前 model 并设置 `currentModel`
   - 返回对象新增: `currentModel`, `changeModel`

2. 创建 `src/app/api/runtime/sessions/[id]/model/route.ts`:
   - PATCH 端点，接收 `{ model: string, provider: string }`
   - 通过 `getSession(id)` 找到 session
   - 验证 model 在该 provider 的 model 列表中（使用 `getModelIdsForProvider`）
   - 更新 `session.model = model`
   - 返回 `{ ok: true, model }`
   - session 不存在 → 404

3. 修改 `src/app/api/runtime/sessions/[id]/chat/route.ts`:
   - 在 `runWebSession` 调用时，将 `session.model` 传入 `config.model`
   - 当前 adapter 只传递 `...options.config`，需要确保 chat route 在调用时传入:
     ```typescript
     await runWebSession(session, {
       query,
       config: { model: session.model },
       onEvent: ...
     });
     ```

4. 修改 `src/components/workspace-shell.tsx`:
   - 导入 `ModelSelector` 组件
   - 替换 sidebar 中 "Model" 信息区（约第 77-83 行的两个 div）为:
     ```tsx
     <ModelSelector
       currentModel={currentModel}
       onModelChange={changeModel}
     />
     ```
   - 从 `useChatSession` 解构 `currentModel` 和 `changeModel`
   - 保留 sidebar 中的 Session 信息区域不变
</action>

## Threat Model

- **Attack vector:** Model 切换端点可被滥用切换到昂贵模型
- **Mitigation:** v1 本地部署，单用户。模型列表硬编码在 `src/utils/model.ts`，不暴露自定义模型端点
- **Risk level:** LOW（本地部署，硬编码模型列表）
