# Plan 02-02: Chat UI + Streaming Rendering

## Objective

改造 Workspace Shell 为可交互的聊天界面：用户输入问题、看到流式回复、查看工具调用状态，形成完整的聊天体验。

## must_haves

- [ ] 用户可以在 Textarea 中输入问题并按 Enter 发送
- [ ] 流式文本逐步渲染（不是等全部完成后一次性显示）
- [ ] 工具调用以简要标签形式展示（工具名 + 状态）
- [ ] 思考状态有明确的视觉指示

<action>
创建 `src/hooks/use-sse-stream.ts`：

```typescript
type SSEEvent = Record<string, unknown> & { type: string };

type SSEStreamCallbacks = {
  onEvent: (event: SSEEvent) => void;
  onError: (error: Error) => void;
  onComplete: () => void;
};

export function consumeSSEStream(
  response: Response,
  callbacks: SSEStreamCallbacks
): void
```

实现逻辑：
1. 获取 `response.body!.getReader()` 和 `new TextDecoder()`
2. 维护一个 `buffer: string` 缓冲区
3. 在 `while (!done)` 循环中：
   - `decoder.decode(value, { stream: true })` 追加到 buffer
   - 按 `\n\n` 分割 buffer，取完整事件
   - 每个 `data: ` 前缀去掉后 `JSON.parse`
   - 调用 `callbacks.onEvent(parsed)`
   - 不完整的部分保留在 buffer
4. 循环结束后调用 `callbacks.onComplete()`
5. catch 错误调用 `callbacks.onError(error)`

导出一个辅助类型：
```typescript
export type ChatMessage = {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  status: 'streaming' | 'complete';
  toolCalls: ToolCallInfo[];
  thinking: boolean;
  thinkingMessage: string | null;
};

export type ToolCallInfo = {
  tool: string;
  status: 'running' | 'done' | 'error';
  args?: Record<string, unknown>;
  result?: string;
  error?: string;
};
```
</action>

<action>
创建 `src/hooks/use-chat-session.ts`：

```typescript
export function useChatSession() {
  // 返回:
  // - messages: ChatMessage[]
  // - isStreaming: boolean
  // - sessionId: string | null
  // - sendQuery: (query: string) => Promise<void>
  // - createSession: () => Promise<void>
  // - error: string | null
}
```

实现逻辑：
1. `useState<ChatMessage[]>([])` 管理消息列表
2. `useState<string | null>(null)` 管理 sessionId
3. `useState(false)` 管理 isStreaming
4. `useState<string | null>(null)` 管理 error

**createSession()**：
- `fetch('/api/runtime/sessions', { method: 'POST' })`
- 解析 `{ sessionId }` 并设置 state
- 将 sessionId 存入 `localStorage.setItem('dexter-session-id', sessionId)`

**sendQuery(query: string)**：
1. 如果没有 sessionId，先 await createSession()
2. 添加 user message 到 messages：`{ id: crypto.randomUUID(), role: 'user', content: query, status: 'complete', toolCalls: [], thinking: false, thinkingMessage: null }`
3. 添加空的 assistant message：`{ id: crypto.randomUUID(), role: 'assistant', content: '', status: 'streaming', toolCalls: [], thinking: true, thinkingMessage: null }`
4. 设置 `isStreaming = true`
5. `fetch(\`/api/runtime/sessions/${sessionId}/chat\`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ query }) })`
6. 调用 `consumeSSEStream(response, callbacks)` 处理流

**SSE 事件处理回调**：
- `thinking`: 更新最后一条 assistant message 的 `thinkingMessage`
- `stream_progress`: 
  - 如果 `mode === 'responding'`，将 `charDelta` 累加到 assistant message 的 content（注意：需要维护一个累积计数器来验证最终 answer）
  - 设置 `thinking = false`
- `tool_start`: 追加到 assistant message 的 `toolCalls: { tool, status: 'running', args }`
- `tool_end`: 更新对应 toolCall 的 `status: 'done'`, `result`
- `tool_error`: 更新对应 toolCall 的 `status: 'error'`, `error`
- `done`: 
  - 设置 assistant message `content = event.answer`（用最终 answer 覆盖累积值，确保准确）
  - 设置 `status = 'complete'`, `thinking = false`
  - 设置 `isStreaming = false`

**重要**：`stream_progress` 的 `charDelta` 是字符增量数，不是实际文本。在 Phase 2 中，前端先用空字符占位，`done` 事件到达后用 `event.answer` 替换为实际内容。这意味着流式体验主要体现在事件到达的节奏感（thinking → tool calls → responding → done），而不是逐字显示。如果想实现逐字效果，需要在 `adapter.ts` 层面修改 `streamAndAccumulate` 来 yield 实际文本增量——但这属于改动 Agent 核心的边界外行为，Phase 2 不做。

实际上，重新审视 `stream_progress` 事件：它只提供 `charDelta`（数字），不提供文本内容。因此 Phase 2 的流式体验策略是：
- 收到 `stream_progress` 时显示"正在生成回复..."的动画
- 收到 `tool_start/end` 时显示工具状态
- 收到 `done` 时一次性显示完整 answer
</action>
