### Task 2: 修复流式消息自动滚动跳动

<read_first>
- src/components/workspace-shell.tsx（包含 scrollIntoView 的 useEffect）
</read_first>

<action>
1. 修改 `workspace-shell.tsx` 中的自动滚动逻辑：
   - 添加 `isNearBottom` 检测函数：
     ```typescript
     const chatContainer = bottomRef.current?.parentElement;
     if (!chatContainer) return;
     const threshold = 150;
     const distanceFromBottom = chatContainer.scrollHeight - chatContainer.scrollTop - chatContainer.clientHeight;
     return distanceFromBottom < threshold;
     ```
   - 将现有的 `useEffect` 改为只在用户位于底部时自动滚动：
     ```typescript
     useEffect(() => {
       const container = bottomRef.current?.parentElement;
       if (!container) return;
       const distanceFromBottom = container.scrollHeight - container.scrollTop - container.clientHeight;
       if (distanceFromBottom < 150) {
         bottomRef.current?.scrollIntoView({ behavior: 'smooth', block: 'end' });
       }
     }, [messages, isStreaming]);
     ```
   - 注意：使用 `container.scrollHeight - container.scrollTop - container.clientHeight` 而非 `isNearBottom` 函数引用，确保 useEffect 内没有闭包陈旧问题
</action>

<acceptance_criteria>
- `workspace-shell.tsx` 的自动滚动 useEffect 中包含 `distanceFromBottom` 计算
- `workspace-shell.tsx` 的 scrollIntoView 仅在 `distanceFromBottom < 150` 时触发
- `bun run typecheck` 通过
</acceptance_criteria>

### Task 3: 工具详情展开过渡动画

<read_first>
- src/components/chat-message.tsx（包含 ToolDetails 组件和 expanded 状态）
</read_first>

<action>
1. 在 `chat-message.tsx` 的 `ToolDetails` 组件中，修改展开区域的过渡效果：
   - 在 `expanded` 条件渲染的 `<div className="space-y-3">` 外层包裹一个容器：
     ```tsx
     <div className={cn(
       'grid transition-all duration-200 ease-in-out',
       expanded ? 'grid-rows-[1fr] opacity-100' : 'grid-rows-[0fr] opacity-0'
     )}>
       <div className="overflow-hidden">
         {/* 原有的 space-y-3 内容 */}
       </div>
     </div>
     ```
   - 这个 CSS grid trick（`grid-rows-[1fr]` / `grid-rows-[0fr]` + `overflow-hidden`）实现了高度从 0 到 auto 的平滑过渡，不需要固定高度值
   - `transition-all duration-200` 确保 200ms 的过渡时间
   - `opacity` 变化配合高度变化提供更自然的视觉效果

2. 将 `<pre>` 标签的 `max-h-[120px]` 改为 `max-h-[200px]`：
   - 工具详情中的参数和返回值 `<pre>` 区域都从 120px 放大到 200px
   - 共有 3 处 `<pre>` 需要修改：参数、返回值、错误
</action>

<acceptance_criteria>
- `chat-message.tsx` 的 ToolDetails 组件包含 `grid-rows-[1fr]` 和 `grid-rows-[0fr]` class
- `chat-message.tsx` 的 ToolDetails 包含 `transition-all duration-200`
- `chat-message.tsx` 中没有 `max-h-[120px]`（全部替换为 `max-h-[200px]`）
- `bun run typecheck` 通过
</acceptance_criteria>
